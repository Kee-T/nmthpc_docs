## Generating Documentation Locally

The documentation can be generated by running `Makefile`. The yaml file `conda_dev_env.yml` file contains the list of requirement to replicate the conda environment. To create the conda environment, [download Anaconda](https://www.anaconda.com/) and then create the environment as follows:
```
conda env create -f conda_dev_env.yml
```
To activate the environment:
```
conda activate read_the_docs
```

Using the `read_the_docs` environment we can generate the documentation as follows:
```
make html
```
The documentation will be stored in the directory `build`, and can be opened in a browser by running:
```
make view 
```
Although not necessary, to cleanup the build folder:
```
make clean
```
